项目全生命周期

从创建到结束，每个阶段都有你要做的事、给 AI 的口令，以及应该写回的文件。

阶段你要做什么给 AI 的口令
0. 创建前 确定项目目标、类型（code/docs/knowledge/automation）和隐私（private-local 用于证件/签证/求职） “用 ide-toolbox 新建一个项目，名字叫 260614-xxx，类型是 knowledge，用途是 xxx，隐私策略是 private-local。”
1. 开始项目 先让 AI 接手，不要直接改文件；检查 AGENTS、ai-context、runbook、suggested-assets “先按 AGENTS.md 接手这个项目。请说明你会读取哪些文件、当前项目状态是什么，不要先改文件。”
2. 开展任务 要求计划、修改、验证、写回；必要时先查询 Agent Library “按长跑模式执行：先给计划，再修改，最后验证。结束前必须更新 ai-context。”
3. 里程碑结束 运行 session-handoff，确保 Current State / Last Session / Open Items 可移交 “这是一个里程碑。请按 session-handoff 收尾，列出已完成、下一步、验证方式、回滚方式。”
4. 晋升资产 只有第二个不同项目仍需要的 SOP/Skill 才晋升；private-local 永不晋升 “评估这次产出是否值得晋升到 Agent Library。若值得，先说明去敏方案。”
5. 项目结束 做 project-health、session-handoff、整理 README/ai-context，再归档预览 “这个项目准备结束/暂停。请做最终收尾：体检、handoff、归档建议、回滚方式。”

必须记住

聊天不是记忆；重要结论写入 docs/ai-context.md 或 docs/YYYYMMDD-*.md。

Notion 项目

新建时同样生成 docs/ai-context.md、docs/HANDOFF.md、docs/suggested-assets.md，旧项目可用 upgrade 补齐。

收尾门禁

有 WARN 先修 WARN，不继续扩功能：./scripts/session-handoff.sh /path/to/project

复利纪律

第一次留项目内；第二次跨项目仍需要才晋升；private-local 永不进共享库。

Agent CLI 自动驾驶

不打开 Cursor 图形界面，也能由 ide-toolbox 在命令行里驱动 Cursor Agent。

命令用途
start只读接手项目，汇报状态、风险和 suggested-assets
plan只读生成计划，列出影响范围、验证、回滚
run --execute受控执行任务，结束后接 health / handoff
milestone里程碑收尾，检查 ai-context 和下一步
new-notion新建 Notion 双轨项目并生成接手 prompt
 
./scripts/agent-cli.sh start "/path/to/project" --dry-run
./scripts/agent-cli.sh plan "/path/to/project" "任务目标" --dry-run
./scripts/agent-cli.sh run "/path/to/project" "任务目标" --execute

默认不使用 --force / --yolo；删除、覆盖、权限、密钥、Git push、Notion 大规模结构修改仍必须确认。
