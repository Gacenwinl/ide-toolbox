## 使用规则

- 会话启动：读 `docs/suggested-assets.md`（若存在）
- 做 Notion 结构前：先查询是否已有 Notion 双轨 / 日历 / 打卡相关 playbook
- 第一次产出：只写回本项目 `docs/` / `content/`
- 第二次跨项目仍需要：去敏后再晋升到 `05_Agent-Library`

## 可晋升内容

- Notion 双轨搭建 SOP
- manifest / NOTION_INDEX 模板
- 日历、打卡、任务库等通用结构模式
- session-handoff 里程碑模板

## 不可晋升内容

- 真实 Notion URL / page_id / data_source_id
- 用户个人资料、健康/签证/求职等实例数据
- GitHub 仓库 URL、Synology 绝对路径
- `private-local` 项目任何内容
